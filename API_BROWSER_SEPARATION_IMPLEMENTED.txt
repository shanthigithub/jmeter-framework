⚙️ CONFIGURATION DETAILS
═══════════════════════════════════════════════════════════════════════════════

API Task Definition (jmeter-api):
  • Family: jmeter-api
  • vCPU: 2 (2048 CPU units)
  • Memory: 4096 MB (4 GB)
  • Log Group: /jmeter/api
  • Use Case: HTTP/REST API load tests
  • Cost: ~$0.09/hour

Browser Task Definition (jmeter-browser):
  • Family: jmeter-browser
  • vCPU: 4 (4096 CPU units)
  • Memory: 8192 MB (8 GB)
  • Log Group: /jmeter/browser
  • Use Case: Selenium/JSR223 browser tests
  • Cost: ~$0.18/hour (2x API)

{

      "testId": "selenium-ui-flow",

      "testScript": "tests/browser-test.jmx",
      "execute": true
    }
  ]
}

Example 3: Mixed Tests
{
  "testSuite": [
    {
      "testId": "api-endpoints",
      "testType": "api",              ← Uses API task (2 vCPU, 4 GB)
      "testScript": "tests/api.jmx",
      "execute": true
    },
    {
      "testId": "ui-automation",
      "testType": "browser",          ← Uses Browser task (4 vCPU, 8 GB)
      "testScript": "tests/ui.jmx",
      "execute": true
    }
  ]
}

🔄 RUNTIME BEHAVIOR
═══════════════════════════════════════════════════════════════════════════════

Lambda Selection Logic (submit-tasks/index.py):

  # Extract testType from test config
  test_type = test.get('testType', 'api')  # Default to 'api'
  
  # Select appropriate task definition
  task_definition = task_def_browser if test_type == 'browser' else task_def_api
  
  # Log selection
  print(f"Test {test_id}: type={test_type}, tasks={num_containers}")
  print(f"Using task definition: {task_definition.split('/')[-1]}")

Lambda Logs Example:
  📊 Test dcp-api-test: type=api, tasks=3
     Using task definition: jmeter-api:12

  📊 Test selenium-test: type=browser, tasks=2
     Using task definition: jmeter-browser:8

💰 COST IMPACT
═══════════════════════════════════════════════════════════════════════════════

Scenario: 100 API tests + 10 Browser tests per day (10 min avg each)

BEFORE (all tests using 2 vCPU / 4 GB):
  • 110 tests × 10 min × $0.09/hour = ~$1.65/day
  • Monthly cost: ~$50

AFTER (right-sized resources):
  • 100 API tests × 10 min × $0.09/hour = ~$1.50/day
  • 10 Browser tests × 10 min × $0.18/hour = ~$0.30/day
  • Total: ~$1.80/day
  • Monthly cost: ~$54

Potential Savings if Browser Tests Were Longer:
  • If browser tests take 30 min instead of 10 min:
    - Browser: 10 × 30 min × $0.18/hour = ~$0.90/day
    - Total: $1.50 + $0.90 = $2.40/day (~$72/month)
  • BEFORE (all 30 min): 110 × 30 min × $0.09/hour = ~$4.95/day (~$148/month)
  • SAVINGS: $76/month (51% reduction)

Key Benefit: Pay for what you need! API tests don't overpay for browser resources.

📊 CLOUDWATCH LOG GROUPS
═══════════════════════════════════════════════════════════════════════════════

New Structure:
  /jmeter/api         ← API test logs (lightweight tests)
  /jmeter/browser     ← Browser test logs (Selenium/JSR223)

Old Structure (deprecated after deployment):
  /ecs/jmeter         ← No longer used after deployment

Benefit: Separate log groups make it easier to:
  • Filter API vs Browser test logs
  • Set different retention policies
  • Monitor resource usage separately
  • Debug issues specific to test type
